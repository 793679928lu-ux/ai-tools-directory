# AETHER 首批原创 SEO 内容页设计

日期：2026-07-06

## 目标

为 AETHER 增加首批可被搜索引擎收录的原创内容页，获取自然搜索流量，并把内容流量引回工具库和商业合作入口。

成功标准：

- 新增 3 个静态内容页：
  - `/guides/ai-writing-tools.html`
  - `/guides/ai-image-tools.html`
  - `/guides/ai-coding-tools.html`
- 首页增加“AI 工具选型指南”入口。
- `sitemap.xml` 包含 3 个内容页。
- 每个内容页有独立标题、描述、canonical、Open Graph 和 JSON-LD。
- 页面内容原创、可读，适合中文用户快速选型。
- 不新增框架、CMS、数据库、后台或第三方脚本。
- 现有测试继续通过，并新增内容页可访问性与 SEO 文件测试。

## 方案选择

### 推荐方案：手写静态内容页

直接新增 3 个 HTML 页面，复用 AETHER 的视觉语言和现有静态部署方式。这样最快、最稳，也方便搜索引擎抓取。

优点：

- 无构建依赖。
- Vercel 可直接托管。
- 页面可被搜索引擎直接读取。
- 适合当前低成本获客阶段。

缺点：

- 后续文章数量变多后，手动维护 sitemap 和页面结构会变麻烦。

### 暂缓方案：接 CMS 或博客系统

CMS 能提升长期内容管理效率，但会引入登录、权限、数据源、模板和部署复杂度。当前只有首批 3 页，不值得引入。

### 暂缓方案：自动生成大量文章

批量生成很多低质量页面可能影响搜索质量，也会损害品牌信任。当前优先做少量可读页面。

## 页面范围

### 首页入口

在工具列表和商业合作区之间新增“AI 工具选型指南”区域。

内容包含 3 张指南卡：

- AI 写作工具推荐：面向写文章、总结资料、营销文案和日常办公。
- AI 绘图工具推荐：面向海报、产品图、概念图和设计灵感。
- AI 编程工具推荐：面向代码补全、项目修改、调试和快速原型。

每张卡链接到对应内容页，并保留 AETHER 的深色科技感样式。

### 内容页结构

每个内容页保持统一结构：

1. 顶部品牌导航：返回首页、工具库、商业合作。
2. Hero：页面主题、适合人群、更新时间。
3. 快速结论：用简短列表告诉用户应该怎么选。
4. 推荐工具：列出 3-5 个相关工具，链接到官网或首页工具库。
5. 选型标准：说明什么场景该选什么工具。
6. 常见问题：覆盖新手最容易问的问题。
7. 转化入口：返回 AETHER 工具库、提交工具、商业合作。

## 内容原则

- 以中文用户的实际使用场景写，不做空泛介绍。
- 不编造价格、折扣、联盟收益或未验证功能。
- 不承诺工具一定免费、永久免费或一定可用。
- 商业入口要明确标注，不伪装成客观排名。
- 内容长度控制在静态页可读范围，不做冗长水文。

## 文件边界

新增：

- `guides/ai-writing-tools.html`
- `guides/ai-image-tools.html`
- `guides/ai-coding-tools.html`

修改：

- `index.html`：增加指南入口。
- `styles.css`：增加首页指南卡和内容页样式。
- `sitemap.xml`：加入 3 个内容页。
- `README.md`：说明如何继续新增指南页。
- `tests/file-compat.test.js`：校验指南页、sitemap 和基础 SEO。

不改动：

- 不改现有工具筛选和收藏逻辑。
- 不接支付。
- 不接 CMS。
- 不新增第三方脚本。
- 不提交任何密钥、cookie、token 或商户凭据。

## 数据流

1. 用户从首页指南卡进入内容页。
2. 搜索引擎通过 sitemap 发现内容页。
3. 内容页通过内部链接引导用户回到首页工具库或商业合作区。
4. 商业咨询仍通过现有邮箱和微信人工处理。

## 测试

自动化测试覆盖：

- 3 个指南 HTML 文件存在。
- 每个指南页包含唯一标题、meta description、canonical 和 JSON-LD。
- 首页包含 3 个指南链接。
- `sitemap.xml` 包含 3 个指南 URL。
- 页面不引入远程脚本或远程样式。
- 现有 13 项测试继续通过。

人工验证覆盖：

- 首页指南卡在桌面和手机端可读。
- 3 个内容页可直接打开。
- 内容页导航能回首页、工具库和商业合作。
- Vercel 线上部署后 3 个内容页均返回 200。

## 非目标

本阶段不做：

- 文章后台。
- 评论系统。
- 自动生成文章。
- 搜索站内文章。
- 多语言版本。
- 访问统计脚本。
- 支付或广告客户自助后台。

## 后续阶段

2. 根据搜索表现优化标题和内部链接。
3. 有流量后再接访问统计，评估哪些内容页带来商务咨询。
4. 内容页流量稳定后，再考虑广告落地页和自助投放流程。
