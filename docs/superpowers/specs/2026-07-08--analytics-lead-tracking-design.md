# AETHER 访问追踪与广告线索闭环设计

## 目标

让 AETHER 能开始判断“有没有人来、点了什么、有没有广告咨询和成交记录”。本次只做静态站可直接上线的低成本闭环，不引入数据库、支付系统、登录系统或自动群发。

## 范围

- 给公开页面接入 Vercel Web Analytics 入口脚本，并记录关键点击事件。
- 在广告合作页增加广告咨询表单，帮助潜在客户生成邮件或复制咨询内容。
- 在获客运营后台增加数据口径说明，以及本机数据导入/导出，方便备份和换设备。
- 保持 `growth.html` 为 `noindex,nofollow`，继续只记录手动发布后的运营数据。

## 数据流

公开页面加载本地 `analytics.js`。线上 Vercel 环境中，它注入 Vercel Web Analytics 脚本；本地或文件打开时不发网络请求，只把关键点击保存在浏览器本机日志中，方便调试。广告咨询表单只生成邮件或复制文本，不把用户填写内容上传到网站后台。

获客运营后台继续使用 `localStorage` 保存发帖、曝光、点击、回复、线索和成交状态。导出会生成 JSON 文件；导入只接受同结构 JSON，并覆盖当前浏览器的后台数据。

## 验证

- 静态页面不引入远程脚本地址，避免本地运行失败。
- 首页脚本加载顺序仍为本地 classic script。
- 获客运营后台仍保持 `growth-data.js` 后接 `growth.js`。
- `node --test tests/*.test.js` 全部通过。
