# AETHER 系统化界面设计

日期：2026-07-05

关联文档：[AI 指南针首期变现设计](./2026-07-05--monetization-design.md)

## 品牌

- 英文名称：`AETHER`
- 副标题：`AI DISCOVERY OS`
- 中文描述：为所有人提供简单、直观的 AI 工具发现系统。

`AETHER` 作为页面主品牌，替换所有“AI 指南针”文字、页面标题和辅助说明。工具名称、分类名称和主要操作继续使用中文，避免为了科技感牺牲理解成本。

## 目标

将现有单页导航改造成类似桌面操作系统的工具发现界面：

- 第一眼具有 macOS/iOS 式的熟悉感和高品质科技感。
- 不同年龄段用户无需说明即可完成搜索、分类和打开工具。
- 动态效果增强空间感和反馈，但不遮挡内容、不拖慢操作。
- 商业合作入口自然融入系统界面，并明确标注广告或赞助。
- 继续使用静态 HTML、CSS 和 JavaScript，不引入前端框架。

成功标准：

- 用户进入页面后能在 5 秒内找到搜索和分类入口。
- 搜索、分类、收藏、工具跳转和商务入口均有文字标签。
- 触控目标不小于 44×44 像素。
- 键盘可以完成搜索、筛选和打开工具。
- 关闭动画或在低性能设备上仍可完整使用。
- 手机端不出现缩小版桌面窗口，而使用原生移动布局。

## 视觉系统

### 背景

- 深蓝黑色基底。
- 紫色、蓝色和少量粉色极光渐变。
- 使用 CSS 渐变与模糊层实现，不加载大型背景图片或视频。
- 背景只缓慢移动，不跟随鼠标快速晃动。

### 玻璃层

- 菜单栏、主窗口、Dock 和浮层使用半透明玻璃效果。
- 内容卡片使用较深的实色底，保证文字对比度。
- 圆角分为 12、18、24 和 30 像素四级，不随意混用。
- 阴影只表达层级，不制造过度发光。

### 品牌图标

- 使用 `AE` 字母组合形成圆角方形图标。
- 主色为紫蓝到青色渐变。
- 不使用苹果 Logo，不复制 macOS 系统图标。
- 分类图标采用简单几何符号，并始终搭配中文文字。

## 桌面端布局

适用宽度：1024 像素及以上。

### 顶部菜单栏

- 左侧显示 AETHER、工具库、分类、收藏和商业合作。
- 右侧只保留装饰性的系统状态信息，不伪装成真实电量或设备状态。
- 菜单栏固定在顶部，滚动时保持半透明。

### 主窗口

主窗口使用 macOS 式分栏结构：

- 左侧为分类侧栏。
- 右侧为搜索、状态摘要、快捷筛选和工具内容。
- 左上角三色圆点只作为视觉语言，不提供关闭网页等虚假操作。

### 分类侧栏

顺序如下：

1. 为你推荐
2. 全部工具
3. 我的收藏
4. AI 对话
5. 图像设计
6. 视频创作
7. 办公效率
8. 编程开发
9. 音频音乐

当前分类使用高亮背景、图标和文字三重反馈，不能只依赖颜色。

侧栏底部放置精简的品牌合作卡，跳转到商业合作区域。

### 搜索

- 搜索框位于内容区顶部，宽度优先。
- 支持工具名称、用途、标签和分类。
- 保留 `⌘/Ctrl + K` 快捷键。
- 快捷键打开类似 Spotlight 的搜索浮层。
- 搜索结果即时更新并显示数量。
- 空状态提供“清除筛选”和“提交工具”两个明确操作。

### 工具卡

- 桌面端默认三列。
- 每张卡包含品牌缩写图标、工具名、简短说明、分类、赞助或精选标识和打开按钮。
- 整张卡可点击，同时保留清晰的外链箭头。
- 悬停时上移 4 像素并增强边框，不旋转、不大幅缩放。
- 赞助卡使用独立配色并显示“赞助”。

### Dock

- 页面底部显示分类快捷 Dock。
- 图标悬停或键盘聚焦时轻微放大并显示中文标签。
- 点击后切换分类并滚动到工具列表顶部。
- Dock 不遮挡页脚、商业合作按钮或手机端内容。

## 移动端布局

适用宽度：小于 768 像素。

- 不显示桌面菜单栏、三色圆点或固定侧栏。
- 顶部显示 AETHER 品牌、搜索按钮和菜单按钮。
- 搜索框在首页首屏内可见。
- 分类使用横向滚动胶囊按钮。
- 工具卡使用单列大卡片。
- 底部使用四项导航：发现、分类、收藏、合作。
- 商业合作价格卡和联系方式纵向排列。
- 所有按钮至少 44 像素高。

768 到 1023 像素之间使用两列工具卡和可折叠侧栏。

## 交互与动态效果

### 页面进入

- 菜单栏、主窗口和 Dock 依次淡入。
- 总时长不超过 600 毫秒。
- 内容可在动画完成前操作，不设置加载遮罩。

### 搜索

- `⌘/Ctrl + K` 打开搜索浮层。
- 输入时工具结果即时筛选。
- 按 `Escape` 关闭浮层并返回原焦点。

### 分类与 Dock

- 侧栏、分类按钮和 Dock 共享同一筛选状态。
- 切换分类只更新工具区域，不刷新页面。
- 当前分类同步高亮。

### 收藏

- 用户可在工具卡上收藏或取消收藏。
- 收藏数据只保存在浏览器 `localStorage`。
- 不要求登录，也不上传收藏数据。
- 本地存储不可用时，收藏按钮隐藏，其他功能不受影响。

### 反馈

- 复制微信号后显示短暂提示。
- 外链打开前不弹出多余确认。
- 邮件按钮生成预填主题和正文。
- 按钮按下、加载和失败状态都必须可见。

### 动效限制

- 使用 transform 和 opacity 为主。
- 不使用持续高频粒子、鼠标追踪光斑或自动滚动。
- 支持 `prefers-reduced-motion: reduce`，禁用背景移动、窗口进入和 Dock 放大。
- 页面失去焦点时暂停非必要动画。

- 商业合作区域展示精选置顶、内容流广告、首页横幅和组合套餐。

## 数据与代码

继续沿用现有文件：

- `index.html`：系统化页面结构、语义区域和商务容器。
- `styles.css`：响应式布局、玻璃视觉、动效和减少动态效果规则。
- `data.js`：工具、分类、广告、商务价格和联系方式。
- `app.js`：搜索、分类、收藏、Dock、邮件生成和微信复制。

不增加构建步骤、第三方 UI 框架或运行时依赖。

状态边界：

- 搜索词和当前分类仅保存在内存。
- 收藏保存在本地浏览器。
- 商务数据来自 `data.js`。
- 网站不保存订单、付款信息或访客联系方式。

## 可访问性与易用性

- 正文对比度至少满足 WCAG AA。
- 不使用低于 12 像素的功能性文字。
- 所有图标按钮有可读名称。
- 焦点样式明显，且不被动画移除。
- 分类切换使用 `aria-pressed` 或等效状态。
- 搜索结果数量使用礼貌的动态播报。
- 对颜色辨识困难用户，同时提供文字和形状提示。
- 老年用户可以通过放大浏览器字体继续操作，布局不溢出。

## 性能

- 首屏不加载大图、视频、外部字体或动画库。
- CSS 背景和图标优先使用渐变、内联 SVG 或文字缩写。
- JavaScript 保持原生实现。
- 避免在滚动事件中反复计算布局。
- 动画只作用于合成层属性。

## 测试

自动化测试：

- 品牌名称、标题和描述均替换为 AETHER。
- 搜索和分类逻辑保持通过。
- 收藏写入、读取和删除逻辑通过。
- 商务配置、价格、邮箱和微信号有效。
- 页面不存在旧品牌名和占位邮箱。
- 页面不依赖远程样式、字体或核心脚本。

人工验证：

- 1440、1024、768、390 和 320 像素宽度。
- Safari、Chrome 和移动端触控。
- 键盘操作、焦点顺序和快捷搜索。
- 减少动态效果模式。
- 搜索、分类、收藏、外链、邮件和微信复制。
- Vercel 线上部署后的主视觉和功能。

## 非目标

本次不实现：

- 真正的多窗口桌面系统。
- 可拖动窗口、桌面文件或右键菜单。
- 登录、跨设备收藏同步或用户账户。
- 伪装成真实操作系统的电量、网络或通知权限。
- 大型 3D 场景、视频背景或持续粒子动画。
- 与工具导航和商业合作无关的系统应用。
